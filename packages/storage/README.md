# @objectos/plugin-storage

ObjectOS native storage abstraction layer. Handles file interactions and key-value persistence across difference backends.

## Overview

The Storage plugin provides a unified API for:

1. **Blob Storage**: Saving files (images, documents) with metadata.
2. **Key-Value Storage**: Simple persistence for configuration or session data.

It follows the "Driver" pattern, allowing you to switch between local disk, S3, or database storage without changing application code.

## Features

- ✅ **Unified API**: Simple `put`, `get`, `del` methods for all backends.
- ✅ **Multiple Drivers**:
  - `memory`: For ephemeral testing.
  - `fs`: Local filesystem storage.
  - `s3` (Planned): AWS S3 compatible object storage.
  - `sqlite`: Key-value storage in SQLite.
- ✅ **Streaming Support**: Handle large files with generic Node.js streams.
- ✅ **Namespace Isolation**: logical separation of storage buckets.
